    App overview

    A simple weather app that fetches and displays current weather data based on city search.


    This weather application serves as a demonstration of my core programming competencies in front-end development, showcasing my proficiency in HTML, CSS, and JavaScript. Beyond foundational web technologies, the project highlights my ability to work with external APIs – specifically how to fetch, process, and dynamically display real-time data. Key technical aspects demonstrated: - Modern Front-End Development: Clean, responsive UI built with semantic HTML and CSS Flexbox/Grid - Asynchronous JavaScript: Implementation of fetch() or axios for API calls with proper error handling - Data Integration: Transformation of JSON API responses into user-friendly visualizations - State Management: Handling loading states, errors, and data updates - User Experience: Intuitive design with location detection and search functionality The app connects to [WeatherAPI/OpenWeatherMap], retrieving and presenting metrics like temperature, humidity, and forecasts while maintaining optimal performance. This project reflects my understanding of full client-side implementation – from UI design to data layer integration."

    Feature

    Search weather by city name.
    Display: Temperature, Conditions(sunny, rainy, cloudy...) Humidity and Wind speed.


    The application features city-based weather search functionality, delivering real-time atmospheric data through a clean, intuitive interface. Upon entering a location, the system displays comprehensive weather metrics including;
    - Temperature: Measured in degrees Celsius (°C) with precision to one decimal place.
    - Humidity: Expressed as a percentage (%) for immediate clarity
    - Weather Conditions: Visually represented through icons and descriptors (sunny, rainy, cloudy, etc.)
    - Wind Speed: Reported in kilometers per hour (km/h) for regional accuracy.
    The interface dynamically updates to reflect current conditions, presenting data in a user-friendly card layout with appropriate visual indicators. Temperature values feature color-coding (e.g., warm hues for higher temperatures, cool tones for lower ranges) to enhance quick interpretation.


    Technical implementation leverages:
    1. API Integration: Robust error handling for invalid city entries
    2. Data Normalization: Consistent unit presentation across all metrics
    3. Responsive Design: Optimized display for mobile and desktop views
    4. Conditional Rendering: Context-appropriate weather icons and background themes
    This implementation demonstrates my ability to transform raw API data into meaningful user experiences while maintaining technical accuracy and visual coherence.

    Technologies used

    Frontend: HTML, CSS and JavaScript
    API: Open weather app

    This weather application was developed using the following technologies and services: Frontend Development - HTML5: Semantic markup for structured and accessible content.
    - CSS3: Responsive styling with Flexbox/Grid for optimal viewing across devices.
    - JavaScript (ES6+): Dynamic data handling, DOM manipulation, and asynchronous API calls.
    API Integration
    - OpenWeatherMap API: Fetches real-time weather data, including temperature, humidity, wind speed, and conditions. - Data Endpoints: Current weather data (/weather) with metric units (Celsius, km/h). - Error Handling: Graceful fallbacks for invalid city entries or API failures. Development Approach
    - Mobile-First Design: Ensures usability on smartphones, tablets, and desktops.
    - Asynchronous Programming: Uses fetch() with async/await for seamless API interactions.
    - User-Centric Features:
    - City-based search with auto-suggestions (optional enhancement).
    - Dynamic icons for weather conditions (sunny, rainy, etc.).
    - Unit consistency (Celsius, %, km/h).
